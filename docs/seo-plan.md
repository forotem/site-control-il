# תוכנית לבניית אתר מקודם SEO (אסטרטגיית Vibe Coding)

תוכנית זו מבוססת על שיטת העבודה המהירה ("Vibe Coding") ליצירת אתר שמדורג גבוה בגוגל תוך שימוש ב-AI (כמו Claude Code) בתוך VS Code.

## שלב 1: מחקר מילות מפתח מבוסס AI (הביקוש)
המטרה: למצוא מה הלקוחות מחפשים בדיוק, ולא להסתמך על ניחושים.

1. **פתיחת טרמינל ב-VS Code:**
   הרץ פקודה ל-AI (או בצ'אט):
   > "אני בונה אתר ל[סוג העסק]. תן לי רשימה של 25-50 מילות מפתח שהלקוחות שלי מחפשים. חלק אותן לקטגוריות: חירום (Emergency), שירותים (Service), בעיות (Problem), ומיקום (Local)."

2. **ניתוח כוונת המשתמש (Intent):**
   זהה אילו מילים מעידות על כך שהלקוח מוכן לקנות *עכשיו* (למשל "תיקון משאית דחוף") ואילו הן רק למחקר. נתמקד בביטויים בעלי כוונת רכישה גבוהה.

### 🎯 דוגמה מעשית: מילות מפתח לעסק צילום ומעקב

**1. ביטויי הליבה (High Intent) – "אין תשתית"**
הלקוחות מחפשים פתרון למקומות ללא חשמל ואינטרנט:
- מצלמות אבטחה סולאריות
- מצלמת אבטחה ללא חשמל
- מצלמת אבטחה עם סים (SIM)
- מצלמות אבטחה ללא אינטרנט / ללא WIFI
- מצלמות שטח על סוללות
- מצלמת אבטחה סלולרית 4G
- מצלמות אבטחה לשטח פתוח

**2. זנב ארוך: אתרי בנייה וקבלנים (B2B) – הכסף הגדול**
- מצלמות אבטחה לאתרי בנייה
- פתרונות שמירה לאתרי בנייה
- ניהול אתר בנייה מרחוק
- מצלמות מנוף / מצלמות לפרויקטים
- תיעוד התקדמות בנייה (Time-Lapse)
- פיקוח על קבלני משנה
- מניעת גניבות ציוד מאתרי בנייה

**3. זנב ארוך: חקלאות ושטחים פתוחים**
- מצלמות נסתרות לשטח חקלאי
- מיגון ציוד חקלאי
- מצלמות למרעה / דיר / רפת
- אבטחת מטעים וכרמים
- מצלמות שביל סלולריות
- מצלמות אבטחה עמידות למים ואבק

**4. הבידול שלך: "גיבוי ענן וביטוח" (מילות מפתח מנצחות)**
- מצלמת אבטחה עם גיבוי לענן
- מצלמת אבטחה נגד גניבה (Anti-theft)
- שמירת צילומים בענן בזמן אמת
- מצלמה שמשדרת לנייד בזמן אמת

**5. שאלות שגולשים שואלים (Perfect for Blog/Content)**
- "איך לאבטח אתר בנייה בלי חשמל?"
- "כמה עולה מצלמת אבטחה סולארית?"
- "פתרונות צילום טיים לאפס לפרויקטים בבנייה"
- "איך למנוע גניבות סולר/ציוד בשטח?"
- "מה ההבדל בין מצלמת שביל למצלמת אבטחה סלולרית?"

## שלב 2: מיפוי מבנה האתר (ההיצע)
בניית דפים ייעודיים לכל שירות ולכל מיקום (Programmatic SEO).

1. **תכנון הראוטינג (Next.js App Router):**
   במקום דף אחד כללי, צור מבנה דינמי בתיקיית `app`:
   - `app/services/[service-slug]/page.tsx` (לדפי שירותים ספציפיים)
   - `app/locations/[city-slug]/page.tsx` (לדפי מיקום)
   - או שילוב: `app/[city]/[service]/page.tsx`

2. **יצירת תוכן עשיר ומקומי:**
   השתמש ב-AI כדי לכתוב תוכן ייחודי לכל דף מיקום.
   > **הנחיה ל-AI:** "כתוב תוכן לדף הנחיתה של [שם העיר]. כלול ציוני דרך מקומיים, בעיות נפוצות באזור זה, ושאלות ותשובות רלוונטיות לאזור."
   *טיפ מהסרטון:* המתחרים "המשעממים" לא עושים את זה. פירוט על מקומות מקומיים עוזר לדירוג.

### 💡 אסטרטגיית קופירייטינג ותמחור לדפי השירות

**כותרת ראשית (H1) שתופסת "זנב ארוך":**
```
"מצלמות אבטחה סולאריות לאתרי בנייה ושטחים חקלאיים – ללא צורך בחשמל או אינטרנט"
```

**שדרוג שמות החבילות (במקום שמות כלליים):**
- **חבילה בסיסית** → "פתרון עצמאי לשטח – התקנה ותמיכה עצמית"
- **חבילה מתקדמת** → "שירות ניהול מצלמות אבטחה וגיבוי לענן – פתרון ראש שקט לקבלנים"
- **חבילה טיים-לאפס** → "הפקת סרטוני Time-Lapse שיווקיים – תיעוד התקדמות פרויקטים בבנייה"

**שורת טאגליין (Tagline) קוויט ומנצח:**
> "🎥 **גנבו לכם את המצלמה? אצלנו התיעוד נשאר.**"  
> (משלב את הכאב המרכזי עם הפתרון הייחודי שלך)

**מיקום מילות מפתח בתוכן:**
- בפסקה הראשונה: הזכר את הבעיה ("ללא תשתית", "גניבות בשטח", "צורך בניטור מרחוק")
- בתוך תיאור החבילות: השתמש בשמות ההיצע (סוללה, סים, ענן, פאנל סולארי)
- בשאלות ותשובות: כתוב Q&A לפי מילות המפתח של השלב 1

## שלב 3: אופטימיזציה טכנית (Technical SEO)
שימוש ב-Claude Code לתיקון ושיפור הקוד הטכני.

1. **ביקורת עומק (Deep Audit):**
   בקש מה-AI:
   > "עבור על כל הקבצים בפרויקט, תנתח את ה-SEO הטכני ותציע רשימת תיקונים לשיפור הדירוג."

2. **יישום Metadata API:**
   ודא שבכל `page.tsx` יש פונקציית `generateMetadata` דינמית שמושכת כותרות (Titles) ותיאורים (Descriptions) המכילים את מילות המפתח שמצאנו בשלב 1.

3. **קבצי ליבה:**
   - צור קובץ `app/sitemap.ts` לייצור מפת אתר דינמית.
   - צור קובץ `app/robots.ts`.
   - הוסף Schema Markup (נתונים מובנים) לעסקים מקומיים (LocalBusiness Schema).

## שלב 4: ביצועים ומהירות (Vitals)
גוגל מדרג גבוה אתרים מהירים.

1. **אופטימיזציית תמונות:**
   - המרת כל התמונות לפורמט WebP.
   - שימוש ברכיב `<Image />` של Next.js עם `priority` לתמונות LCP.
   
2. **בדיקת PageSpeed:**
   הרץ בדיקה ב-Google PageSpeed Insights. אם יש שגיאות, העתק אותן והדבק ב-VS Code ל-AI:
   > "תקן את שגיאות הביצועים האלה: [הדבק שגיאה]."

## שלב 5: סוכני משנה (Sub-Agents)
בזמן שאתה עובד על פיצ'ר אחד, הפעל "סוכנים" ברקע דרך הטרמינל/צ'אט:

- **סוכן 1:** "עבור על כל התמונות בפרויקט `public/` וודא שיש להן תגיות alt מתאימות ל-SEO."
- **סוכן 2:** "בדוק שכל הקישורים הפנימיים (Internal Links) בין דפי השירות והמיקום תקינים והגיוניים."
- **סוכן 3:** "צור קובץ `llms.txt` שיעזור למנועי AI (כמו ChatGPT/Perplexity) לקרוא ולהבין את האתר שלי טוב יותר."
